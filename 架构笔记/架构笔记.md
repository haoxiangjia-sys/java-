and t1( w1 , x, y );  --图下方的第一个 AND 门。它计算如果x和y都是 1，是否会产生进位，结果存在 w1。

ci (Carry-in): 来自低位的进位。
s (Sum): 当前位的加法结果。
co (Carry-out): 产生给高位的进位。    
net：wire w0, w1, w2
instance name: t0, t1, t2, t3, t4

1：逻辑 1 / 高电平 / True。
X：未知状态 (Unknown)。通常出现在未初始化的寄存器，或者两个不同的信号同时驱动一根导线（冲突）时。

ISA(instruction set architecture) define: wordside , instruction , state(eg: registers, memory) , execution model

All instructions are 8 bit.(4 bits opcode: representing an operation.)(4 bit operand: representing data.)
--$2^4 = 16$，意味着这个处理器最多只能识别 16 种不同的动作（如加法、跳转等）。

The processor state is represented by four 8-bit registers.
pc (Program Counter)
oreg (Operand Register)
areg(Accumulator A) :Left-hand operand and result of arithmetic.
breg (Register B):Right-hand operand.

The execution cycle has 3-stages: Fetch(从内存中读取 pc 指向的指令。), 
Increment PC(pc = pc + 1), 
Execute译码并完成操作（例如让 areg 加 breg）.

Four classes of instruction:
▶ Data access: 9 instructions.  读写内存，寄存器之间搬运数据
▶ Branching, jumping and calling: 4 instructions.  实现 if 语句、循环和函数调用
▶ Expression evaluation: 2 instructions.  数学运算
▶ Prefix (special): 1 instruction.

nine of the sixteen instructions:
ADD |   areg ← areg + breg          | add registers and store in areg
SUB |   areg ← areg − breg          | subtract registers and store in areg
LDAC| areg ← oreg                   | load constant into areg
LDBC| breg ← oreg                   | load constant into breg
LDAP| areg ← pc + oreg              | load address in program
BR  | pc ← pc + oreg                | branch relative unconditional
BRZ |if areg = 0 then pc ← pc + oreg| branch relative zero
BRN |if areg < 0 then pc ← pc + oreg| branch relative negative
BRB |pc ← breg                      | branch absolute

ISA State: Registers
寄存器是 CPU 内部极快的小型存储单元。它们分为两大类：
通用寄存器 (General-purpose registers)： 软件可以自由控制，用于执行典型的指令（如加法、搬运数据） 。一组通用寄存器被称为寄存器堆 (Register file) 。
专用寄存器 (Special-purpose registers)： 具有特定功能，通常只能通过特殊指令访问 。常见的例子包括：
程序计数器 (PC)： 存放下一条待执行指令的地址 。
指令寄存器 (Instruction Register)： 存放当前正在执行的指令 。
状态寄存器 (Status Register)： 记录处理器状态（如计算结果是否为负、是否溢出），可影响程序分支 。
堆栈指针 (Stack Pointer)： 指向内存中栈顶的地址 。

ISA State: Memory
(Address space) 定义了程序可以访问的内存地址范围 。
addressability | How many bits do we need to access elements of memory?
granularity    | What does each address refer to? E.g., byte-addressible if each element of the address space is a byte (8-bit word).
alignment      | For n-byte word at address x, if x ≡ 0 mod n then x is aligned. Is unaligned access allowed?
endianness     | For an n-byte word, what order do we store the bytes in each word. E.g., w = ⟨w0,w1, . . . ,wn−1⟩
little-endian  | “left-to-right”, with the first bytes stored at lower memory addresses.
big-endian     | “right-to-left“, with the last bytes stored at lower memory addresses.


1. 指令的结构与语义 (Structure & Semantics)
指令是程序员控制处理器的基本单位，改变着机器的“状态”。每一条指令通常包含两个核心部分：
操作码 (Opcode)： 决定执行什么操作（例如：加法、读取内存）。
操作数 (Operand)： 决定操作的对象。操作数可以是立即数（硬编码的数值）、寄存器或内存地址。
语义定义： 描述指令具体的行为。例如 add r0, r1, r2 的语义是 GPR[0]←GPR[1]+GPR[2]。

2. 指令编码：固定长度 vs 变长
设计 ISA 时，最重大的决策之一是如何在内存中表示这些指令：
特性	固定长度编码 (Fixed Length)	变长编码 (Variable Length)
定义	所有指令长度相同（如 MIPS 是 32-bit）	指令长度根据需求变化（如 x86 是 1-15 字节）
优点	易于取指和译码，硬件设计简单	代码密度高，节省存储空间
缺点	浪费空间（代码密度低），编码压力大	硬件译码极其复杂（不知道下一条指令从哪开始）
例子	Hex 8 (8-bit), MIPS32	Intel IA-32 (x86)
