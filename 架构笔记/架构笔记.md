pc (Program Counter)
oreg (Operand Register)

The execution cycle has 3-stages: Fetch(从内存中读取 pc 指向的指令。),

Execute译码并完成操作（例如让 areg 加 breg）.

▶ Prefix (special): 1 instruction.

ISA State: Registers
寄存器是 CPU 内部极快的小型存储单元。它们分为两大类：
通用寄存器 (General-purpose registers)： 软件可以自由控制，用于执行典型的指令（如加法、搬运数据） 。一组通用寄存器被称为寄存器堆 (Register file) 。
专用寄存器 (Special-purpose registers)： 具有特定功能，通常只能通过特殊指令访问 。常见的例子包括：
程序计数器 (PC)： 存放下一条待执行指令的地址 。
指令寄存器 (Instruction Register)： 存放当前正在执行的指令 。
状态寄存器 (Status Register)： 记录处理器状态（如计算结果是否为负、是否溢出），可影响程序分支 。
堆栈指针 (Stack Pointer)： 指向内存中栈顶的地址 。

ISA State: Memory
(Address space) 定义了程序可以访问的内存地址范围 。
addressability | How many bits do we need to access elements of memory?
granularity    | What does each address refer to? E.g., byte-addressible if each element of the address space is a byte (8-bit word).
alignment      | For n-byte word at address x, if x ≡ 0 mod n then x is aligned. Is unaligned access allowed?
endianness     | For an n-byte word, what order do we store the bytes in each word. E.g., w = ⟨w0,w1, . . . ,wn−1⟩
little-endian  | “left-to-right”, with the first bytes stored at lower memory addresses.
big-endian     | “right-to-left“, with the last bytes stored at lower memory addresses.


1. 指令的结构与语义 (Structure & Semantics)
指令是程序员控制处理器的基本单位，改变着机器的“状态”。每一条指令通常包含两个核心部分：
操作码 (Opcode)： 决定执行什么操作（例如：加法、读取内存）。
操作数 (Operand)： 决定操作的对象。操作数可以是立即数（硬编码的数值）、寄存器或内存地址。
语义定义： 描述指令具体的行为。例如 add r0, r1, r2 的语义是 GPR[0]←GPR[1]+GPR[2]。

2. 指令编码：固定长度 vs 变长
设计 ISA 时，最重大的决策之一是如何在内存中表示这些指令：
特性	固定长度编码 (Fixed Length)	变长编码 (Variable Length)
定义	所有指令长度相同（如 MIPS 是 32-bit）	指令长度根据需求变化（如 x86 是 1-15 字节）
优点	易于取指和译码，硬件设计简单	代码密度高，节省存储空间
缺点	浪费空间（代码密度低），编码压力大	硬件译码极其复杂（不知道下一条指令从哪开始）
例子	Hex 8 (8-bit), MIPS32	Intel IA-32 (x86)

两种查找模式（addressing modes）
决定了指令如何解释操作数（Operands），即 CPU 如何找到它需要处理的数据 。
$GPR[i]$：指代第 $i$ 个通用寄存器 
$MEM[i]$：指代内存中地址为 $i$ 的位置 。

Immediate Mode
定义：操作数就是数据本身 。
ARM 语法：使用 # 前缀（如 #1）
例子：add r0, r1, #1 等同于 $GPR[0] <- GPR[1] + 1$ 。

direct mode
定义：操作数是数据的地址 。
寄存器直接寻址：数据就在寄存器中（如 ARM 中的 r1） 。
内存直接寻址：指令直接给出内存地址 。

indirect mode
定义：操作数是“地址的地址”，即寄存器里存的是内存门牌号 。
ARM 语法：使用中括号 []（如 [r1]） 。
逻辑：add r0, [r1], #1 等同于 $GPR[0] <- MEM[GPR[1]] + 1$ 。

Indexed Mode
定义：通过 基址 + 偏移量 (Address + Offset) 来计算最终地址 。
应用场景：访问结构体（Struct）成员 。
ARM 语法：ldrb r2, [r0, #1]（基址寄存器 r0 + 偏移量 1） 。

Auto-indexed Mode
定义：在变址寻址的基础上，访问完数据后自动更新基址或偏移量 。
应用场景：循环遍历数组 。
ARM 术语：称为“后变址”（Post-indexed），如 ldrb r2, [r1], #1 表示先取值，再将 r1 加 1 。
------------------------------------------------------------------------------------
Contorl Flow
定义：描述指令执行顺序的术语，本质上是程序计数器 (PC) 的更新方式 。
隐式更新 (Implicit)：作为执行周期（Fetch-Decode-Execute）的一部分自动发生，通常指向下一条相邻指令 。
显式更新 (Explicit)：通过特定指令（如分支/跳转）修改 PC 寄存器的值 。
应用场景：实现高级语言中的 if 条件判断、while/for 循环以及函数调用 。

处理器硬件实现：
控制通路 vs 数据通路数据通路 (Datapath)：更新处理器状态的硬件集合，包含寄存器、ALU（算术逻辑单元）、内存及其连接线路 。
控制通路 (Control path)：根据指令发出控制信号，决定数据通路中哪些部分处于活动状态，使电路具备“可编程性” 。

控制单元 (Control Unit) 的两种实现 ：
硬连线控制 (Hardwired)：由固定的逻辑门（如与门、或门）实现，速度快但硬件一旦制造便无法更改 。
微程序控制 (Micro-programmed)：将复杂指令分解为一系列底层“微指令”并存入 ROM 。
优点：设计更易修改，可通过更新固件修复硬件漏洞（如针对 Spectre/Meltdown 的补丁） 。

pipeline and Hazards

A typical RISC architecture operates in these five stages:
Fetch Get an instruction from memory.
Decode Decode instruction and read operands from registers.
Execute Execute instruction or calculate memory addresses.
Memory Access memory (loads/stores).
Writeback Save results back to the registers.
流水线气泡 (Bubbles/Stalls)：由于某些阶段延迟或冲突导致的流水线停顿 。

冒险Hazards会导致流水线停顿，使得下一条指令无法在下一个周期执行。
结构冒险 (Structural Hazards)：两条指令同时竞争同一硬件资源。
控制冒险 (Control Hazards)：由分支指令引起。由于不确定下一条取指地址，可能导致取错指令并丢弃，产生气泡 。  （解决方法：使用分支预测器预测结果以维持流水线流动。）
数据冒险 (Data Hazards)：不同阶段的指令修改/读取相同数据。
1，RAW (写后读)：真相关。第二条指令在第一条指令写回前就尝试读取。
2，WAR (读后写)：反相关。第二条指令在第一条读取前就更新了数据。
3，WAW (写后写)：输出相关。多条指令更新同一数据，顺序决定最终结果。

## [cite_start]1. 存储器的基本需求 [cite: 8]
[cite_start]在冯·诺依曼架构中，程序和数据都存储在内存中 [cite: 9]。
* [cite_start]**低延迟**：处理器访问存储器以获取操作指令，必须保证速度够快，避免处理器空转等待 [cite: 10, 11]。
* [cite_start]**大容量**：需要足够的空间来存放程序和海量数据 [cite: 12, 18]。
* [cite_start]**低成本**：制造价格必须低廉 [cite: 12, 20]。
* [cite_start]**寄存器扩展**：由于处理器寄存器数量较少，需要更大的内存空间支持存储 [cite: 21]。

## [cite_start]2. 存储技术对比 [cite: 27]
[cite_start]我们可以使用不同的技术构建存储器，如 Flip-flops、SRAM、DRAM（及 3D 堆叠 DRAM）、Flash、硬盘和磁带 [cite: 28, 29, 30, 31, 32, 33, 34]。

### [cite_start]SRAM vs. DRAM 核心差异 [cite: 39]

| :--- | :--- | :--- |

| **密度** | [cite_start]密度低（6 晶体管/单元） [cite: 43] | [cite_start]密度高（1 晶体管+1 电容/单元） [cite: 49] |

| **刷新** | [cite_start]无需刷新 [cite: 44] | [cite_start]需要刷新 [cite: 51] |

## [cite_start]3. 存储层次的权衡 (Trade-offs) [cite: 84]
存储设计存在不可避免的权衡：
* [cite_start]**越大越慢**：寻找特定位置所需的时间随着容量增加而变长 [cite: 86]。
* [cite_start]**越快越贵**：由于技术限制，高速度意味着高成本 [cite: 86]。
* [cite_start]**高带宽昂贵**：需要更多的布线支撑 [cite: 87, 88]。

### [cite_start]性能参考 (以 2.2 GHz 处理器为例) [cite: 67]
* [cite_start]**SRAM**: 延迟约 1-50 个周期（10^3-10^6 字节） [cite: 63]。
* [cite_start]**DRAM**: 延迟约 200 个周期（约 100 纳秒） [cite: 64]。
* [cite_start]**硬盘**: 延迟达数十毫秒（TB 级别） [cite: 65]。

## [cite_start]4. 缓存机制 (Caches) [cite: 92]
[cite_start]由于主存（DRAM）访问延迟高达约 200 个周期，处理器需要缓存来加速数据请求 [cite: 93, 98]。



### [cite_start]多级缓存示例 (Skylake 架构) [cite: 114]
* [cite_start]**L1 Cache**: 4 周期延迟，32 KB（每核心私有） [cite: 114, 115]。
* [cite_start]**L2 Cache**: 12 周期延迟，1 MB（每核心私有） [cite: 114, 115]。
* [cite_start]**L3 Cache**: 约 44 周期延迟，1.375 MB（所有核心共享资源） [cite: 114]。

## [cite_start]5. 存储层次结构全景 [cite: 120]
1. [cite_start]**寄存器 (Registers)**: 基于触发器，小而极快 [cite: 120]。
2. [cite_start]**缓存 (Caches)**: 基于 SRAM，较小且快 [cite: 120]。
3. [cite_start]**主存 (Main Memory)**: 基于 DRAM，较大且慢 [cite: 120]。
4. [cite_start]**磁盘 (Disk/Flash/Tape)**: 容量巨大且极慢 [cite: 120]。

### [cite_start]实践：Trinity 超级计算机 [cite: 122]

* [cite_start]**硬盘**: 78 PB 文件系统 [cite: 126]。
* [cite_start]**磁带**: 超过 100 PB 长期存储 [cite: 127]。
